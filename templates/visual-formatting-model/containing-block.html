Visual Formatting Model

A containing block

The visual formatting model transforms an element of a Web document into a box. Various properties can be added to the box in order for it to act as a robust base for other boxes.

The W3C documentation demonstrate that the behaviour of child elements are easier to control if particular properties of the containing block are declared. Using a combination of box-sizing, layout mode and a positioning scheme, we find that child elements align and structure themselves in particular configurations depending on the type of layout mode, positioning scheme and box model.

The combination of all three properties set a particular structural context for the child elements to interact within.

Layout mode, positioning scheme and box model are not tangible presentational properties. In order to visualise these properties, the :before pseudo element of three different classes, have been assigned one of the three properties in combination with a particular background color in order for the audience to confirm the existence of the non tangible property.

.border-box-model, .border-box-model *, .border-box-model *:before, .border-box-model *:after { box-sizing: border-box; } .border-box-model:before { content: ''; background-color: rgba(255, 1, 67, 0.6); position: absolute; z-index: -1; top: 50%; right: 0; bottom: 0; left: 0; transform: translate(-0, -50%); border-radius: 10px; } <p class="border-box-model"></p>
